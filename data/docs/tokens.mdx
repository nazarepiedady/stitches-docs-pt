### Definindo objetos simbólicos

Tu podes definir os teus simbólicos como parte da [configuração](/docs/installation#create-your-config-file) de `createStitches`. Há catorze tipos de objetos simbólicos disponíveis.

```jsx
export const { styled, css } = createStitches({
  theme: {
    colors: {
      gray500: 'hsl(206,10%,76%)',
      blue500: 'hsl(206,100%,50%)',
      purple500: 'hsl(252,78%,60%)',
      green500: 'hsl(148,60%,60%)',
      red500: 'hsl(352,100%,62%)',
    },
    space: {
      1: '5px',
      2: '10px',
      3: '15px',
    },
    fontSizes: {
      1: '12px',
      2: '13px',
      3: '15px',
    },
    fonts: {
      untitled: 'Untitled Sans, apple-system, sans-serif',
      mono: 'Söhne Mono, menlo, monospace',
    },
    fontWeights: {},
    lineHeights: {},
    letterSpacings: {},
    sizes: {},
    borderWidths: {},
    borderStyles: {},
    radii: {},
    shadows: {},
    zIndices: {},
    transitions: {},
  },
});
```

> Isto é baseado na [Especificação de Tema de UI do Sistema](https://system-ui.com/theme) por Brent Jackson.

### Utilizando objetos simbólicos

Depois dos teus simbólicos estiverem definidos, podes utilizá-los para estilizar os componentes.

<Preview>
  <DemoButton color="violet" />
</Preview>

```jsx
const { styled } = createStitches({
  theme: {
    colors: {
      __violet800__: 'hsl(252 62% 54.9%)',
    },
  },
});
const Button = styled('button', {
  backgroundColor: '__$violet800__',
});

() => <Button>Button</Button>;
```

Os simbólicos também funcionam dentro da propriedade `css`.

```jsx
const Button = styled('button', {});

() => (
  <Button
    __css__={{
      backgroundColor: '__$violet800__',
    }}
  >
    Button
  </Button>
);
```

Os objetos simbólicos funcionam até com propriedades de CSS abreviada.

```jsx
const Button = styled('button', {
  margin: '$1 $2',
  border: '1px solid $violet800',
});

() => <Button>Button</Button>;
```

### Pseudónimos de objeto simbólico

Tu podes criar simbólicos semântico ao referenciar os simbólicos na mesma escala.
Como sempre, utilize o `$` para referir-se a um simbólico.

```jsx
createStitches({
  theme: {
    colors: {
      __gray500__: 'hsl(206,10%,76%)',
      primary: '__$gray500__',
    },
  },
});
```

### Objetos simbólicos isolados localmente

Tu podes criar um simbólico diretamente dentro de um objeto de estilo ao prefixá-lo com dois sinais de dólar (`$$`).

```jsx
const Button = styled('button', {
  $$shadowColor: 'red',
  boxShadow: '0 0 0 15px $$shadowColor',
});
```

### Objetos simbólicos de escala prefixada

Tu podes pegar um simbólico a partir de quaisquer das tuas escalas de [tema](/docs/tokens) disponível ao prefixá-los com o nome da escala.

```jsx
const Button = styled('button', {
  // aplicar um simbólico de cor para um simbólico isolado localmente
  $$shadowColor: '$colors$purple500',
  boxShadow: '0 0 0 15px $$shadowColor'

  // Utilize um simbólico da escala de tamanhos
  marginTop: '$sizes$1'
})
```

### Convenção de nomenclatura

Nós recomendamos a utilização da caixa de camelo (ou *camel case*), caixa de pascal (ou *pascal case*) ou caixa de serpente (ou *snake case*) para os teus objetos simbólicos de tema. Outros separadores de palavra podem não funcionar como o esperado.

```json
// recomendado
tokenName
token_name
token-name

// evitar
token.name
token$name
token*name
```

Tu podes ler mais a respeito [disto aqui](https://github.com/stitchesjs/stitches/issues/493#issuecomment-801241038).
