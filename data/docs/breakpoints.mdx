---
title: Pontos de Quebra (breakpoints)
description: Definir pontos de quebras responsivo e reutilizáveis.
---

Defina as tuas regras arroba de media no objeto `media`. Tu podes adicionar os quantos que precisares.

```jsx line=2-6
export const { styled, css } = createStitches({
  media: {
    bp1: '(min-width: 640px)',
    bp2: '(min-width: 768px)',
    bp3: '(min-width: 1024px)',
  },
});
```

> Nota: Escolha a conversão de nomenclatura que preferires. Nós recomendados <Code>bp1</Code>, <Code>bp2</Code>, <Code>bp3</Code> etc. ou <Code>sm</Code>, <Code>md</Code>, <Code>lg</Code> etc.

### Variantes adaptáveis

Uma vez configurado, podes aplicar diferentes variantes em diferentes [pontos de quebra](/docs/breakpoints).

No exemplo abaixo, nós aplicamos a variante da cor `violet` na condição `bp2`.

```jsx line=4-9,12-16
const Button = styled('button', {
  // estilos de base

  variants: {
    color: {
      __violet__: {},
      gray: {},
    },
  },
});

() => (
  <Button
    color={{
      __'@bp2'__: __'violet'__
    }}
  >
    Button
  </Button>
);
```

> Nota: Se estiveres utilizando TypeScript, as tuas variantes serão tipadas.

### Definindo uma variante inicial

Tu deves utilizar o ponto de quebra <Code>@initial</Code> para declarar a variante inicial antes de quaisquer pontos de quebra serem aplicados.

No exemplo abaixo, nós aplicamos inicialmente o `gray`, depois da variante da cor `violet` na condição `bp2`.

```jsx line=4-9,12-16
const Button = styled('button', {
  // estilos de base

  variants: {
    color: {
      violet: {},
      __gray__: {},
    },
  },
});

() => (
  <Button
    color={{
      __'@initial'__: __'gray'__,
      '@bp2': 'violet'
    }}
  >
    Button
  </Button>
);
```

### Utilizando pontos de quebra nos objetos de estilo

```jsx line=4-6
const Button = styled('button', {
  // estilos de base

  '@bp1': {
    // Estilos para bp1 (ponto de quebra número 1)
  },
});
```

> Nota: Na maioria dos casos, nós não recomendamos a aplicação de pontos de quebra adaptáveis em linha. Idealmente, os estilos do teu componente devem ser imutáveis.
