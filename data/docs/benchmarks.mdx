## Os testes

Nós juntamos um conjunto de teste para testar uma gama de fatores desempenho através de várias bibliotecas de CSS-em-JS ou CSS-in-JS.

Todos os testes utilizam notação de objeto. Todos os testes foram executados em um ambiente de produção, em uma maquina com as seguintes especificações:

- Macbook Pro 2019
- 2.3Ghz 8-core i9
- 32GB 2667 MHz DDR4

Tal como a Stitches, nossos [testes de ponto de referência](https://github.com/stitchesjs/stitches-bench) também são de código-aberto.

### Atualizando variantes

A mudança de variantes agora está mais rápida do que nunca. Visto que as variantes podem ser aplicadas condicionalmente, nós garantimos que isto aconteça o mais rápido possível.

{

<BenchmarkChart
  data={[
    { name: 'Stitches', value: 1.3 },
    { name: 'Emotion', value: 2.08 },
    { name: 'styled-components', value: 2.91 },
  ]}
/>
}

> Nota: Visto que a `styled-components` e a `Emotion` não oferecem uma API de variante de primeira classe, isto foi feito através da interpolação de propriedade.

### Atualizando a propriedade `css`

A propriedade `css` é útil para definição de estilos na camada de consumo. Estes estilos precisam ser convertidos em regras de CSS e injetados no DOM. Isto precisa acontecer rápido - e acontece!

{

<BenchmarkChart
  data={[
    { name: 'Stitches', value: 1.06 },
    { name: 'Emotion', value: 2.15 },
    { name: 'styled-components', value: 2.47 },
  ]}
/>
}

> Nota: Visto que a `styled-components` e a `Emotion` não oferecem uma API de propriedade `css` de primeira classe sem a necessidade do Babel, isto foi feito através da interpolação de propriedade.

Neste teste, nós interpretamos o Triângulo Sierpinski. Cada nó é de uma cor diferente, para garantir que os estilos são dinâmicos.

## Sumário

Nossos pontos de referências comparam-se contra a `styled-components` e a `Emotion` porque são as bibliotecas de CSS-em-JS mais populares, elas nos levaram a melhorar o nosso desempenho. Nosso objetivo com estes pontos de referências é fornecer comparações de desempenho transparentes para a comunidade.

Se gostarias que nós incluíssemos a tua biblioteca de CSS-em-JS, ou achas que os testes poderiam ser melhorados, faça o favor de [abrir um questão](https://github.com/stitchesjs/stitches-bench).
