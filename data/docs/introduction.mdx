A Stitches é uma leve, biblioteca de estilização de excelente desempenho com um foco na arquitetura de componente e experiência do programador.

---

## Desempenho

### Tempo de execução quase zero

Na Modulz, o desempenho é primordial. Assim quando estávamos desenhando a Stitches, o desempenho foi a prioridade máxima.

A Stitches evita interpolações de propriedades desnecessárias em tempo de execução, tornando-a significativamente de maior desempenho do que outras bibliotecas de estilização.

Ambas bibliotecas `@stitches/core` e `@stitches/react` pesam em torno de ~6kb compactado em gzip.

### Interpretação no Lado do Servidor

Muitas bibliotecas de UI documentam que variantes responsivas não funcionam com interpretação no lado do servidor. Desde que abstrações populares como a [Next.js](https://nextjs.org/) e [Gatsby](https://www.gatsbyjs.com/) começaram a priorização da SSR, nós quisemos uma solução que funcionasse bem.

A Stitches suporta [interpretação no lado do servidor](/docs/server-side-rendering) de navegador transversal (cross-browser), até para estilos e variantes responsivas.

## Funcionalidades Principais

### Variantes

A Stitches introduz [variantes](/docs/variants) como um cidadão de primeira classe, assim podes desenhar APIs de componente constituível.

Tu podes definir uma única variante, várias variantes, e até [variantes compostas](/docs/variants#variantes-compostas) que te permitem definir estilos com base em uma combinação de variantes.

As variantes podem ser aplicadas condicionalmente ou responsivamente.

### Simbólicos

Uma das funcionalidades mais entusiasmante é a habilidade de definir os teus próprios [simbólicos (tokens)](/docs/tokens) e aplicá-los continuamente como valores de CSS. As propriedades de CSS são mapeadas automaticamente para escalas simbólica. Os simbólicos atá podem ser utilizados em propriedades de CSS abreviadas.

### Criação de Tema

A Stitches fornece uma experiência simples de [criação de tema](/docs/theming) fora da baixa. Crie quantos temas precisares, e aplique-os onde quer que tu quiseres. Cada tema gera um nome de classe de CSS que sobrepõe os simbólicos predefinidos.

```jsx
const darkTheme = createTheme({
  colors: {
    hiContrast: 'hsl(206,2%,93%)',
    loContrast: 'hsl(206,8%,8%)',

    gray100: 'hsl(206,8%,12%)',
    gray200: 'hsl(206,7%,14%)',
    gray300: 'hsl(206,7%,15%)',
    gray400: 'hsl(206,7%,24%)',
    gray500: 'hsl(206,7%,30%)',
    gray600: 'hsl(206,5%,53%)',
  },
  space: {},
  fonts: {},
});
```

### Utilitários

Os [utilitários](/docs/utils) permitem-te criar as tuas próprias propriedades de CSS personalizadas, com a tua própria API.

Utilize os utilitários para criar poderosas combinações de propriedades, pseudónimo de propriedade e introduzir restrições quando necessário.

```jsx
export const { styled, css } = createStitches({
  utils: {
    marginX: (value) => ({
      marginLeft: value,
      marginRight: value,
    }),
    marginY: (value) => ({
      marginTop: value,
      marginBottom: value,
    }),
  },
});
```

### Variantes Responsiva

A Stitches deixa-te configurar [pontos de quebra](/docs/responsive-styles) e aplicar variantes responsivamente utilizando a propriedade `css`. As regras arroba de CSS também podem ser utilizadas em lina se necessário. 

```jsx
export const { styled, css } = createStitches({
  media: {
    bp1: '(min-width: 640px)',
    bp2: '(min-width: 768px)',
    bp3: '(min-width: 1024px)',
    bp4: '(min-width: 1280px)',
  },
});
```

### Sobreposição

A Stitches fornece uma propriedade `css` para sobreposição de estilos em qualquer componente de Stitches. Tu podes utilizar esta propriedade para definir preocupações de disposição e ou [sobreposição](/docs/overriding-styles). Os simbólicos e pontos de quebra também funcionam na propriedade `css`.

```jsx
<Button css={{ backgroundColor: 'red' }}>Button</Button>
```

A Stitches também fornecem uma propriedade `as` polimórfica para definir qual marcador um componente interpreta.

```jsx
<Text as="h1">Heading</Text>
<Text as="p">Paragraph</Text>
```

### Experiência de Programação

Um dos nossos principais objetivos é fornecer a melhor experiência de programador possível. A Stitches fornece uma API completamente tipada, assim quando estiveres utilizando a TypeScript, as propriedades de CSS, os valores, e pontos de quebras serão auto-concluídos por ti.

Além disso, quando estiveres utilizando a propriedade `as`, as sugestões de propriedade se atualizarão.
